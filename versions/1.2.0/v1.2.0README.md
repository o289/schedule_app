# version 1.2.0

## 機能追加

**1. 登録済み日時リストの中で、時刻を編集できるようにする**

目的:
この日にちだけ時刻が違うというものに対応するため

要件:
・Dates モーダルに、時刻ピッカーを導入後、更新ロジックを導入する
・フロントエンドの変更だけで完結

UX 仕様:
・Dates モーダルを時刻を表示しているところをピッカー

画面仕様:

フロー
![モーダル画面](dates-modal-item.jpg)

効果:
登録作業の時間短縮、UX 向上

**2. スケジュールの編集の際に dates に含まれている日時をぞれぞれ、selectedDates,start,end に配置**

目的: 元々の値を把握し、必要ない日時を削除するため
変更: scheduleForm.jsx で使用している、DaySelectDisplay の selectedDates 引数に渡す値を dates(すでに登録されている日たち) にする,start と end は、配列の中身で最も多い組み合わせのものを適用する
効果: UX の向上

**3. CURD の際の自前アラート**

目的: ユーザーがデータの作成・編集・削除に成功したか失敗したか、失敗したならなぜかをアナウンスするため
変更: 共通のコンポーネントを作成し、app.tsx にコンポーネントを配置する。またこれらのの API にコンポーネントに表示する文字列を生成するロジックを追加する
効果: UX 向上

**4. アップデート告知機能**

目的: ユーザーにどのような変更を行なったかを告知するため
変更: 専用ページを作成、新エンドポイントを追加しリリースノートを取得し表示するロジックを作成。取得元はリードミーを予定している
効果: ユーザーの安心感向上
